# Цель
С нуля собрать минимальную стабильную привязку «планета + пешки», убрав текущую ломаную логику якорения пешек.

# Контекст
- Входные данные: требование заказчика на clean-room milestone (без DNA/еды/размножения/автомутаций).
- Ограничения: сохранить существующий оконный цикл, ImGui-панель, базу OpenTK-рендера и структуру проекта.
- Ожидаемый артефакт: единый PlanetSurface + 20 пешек, всегда стоящих на видимой поверхности.

# План
1. Ввести единый PlanetSurface с API: GetHeight/GetSurfacePoint/GetSurfaceNormal/GetSurfaceRadius.
2. Упростить Pawn/PawnAgent до минимального режима без лишних подсистем.
3. Переключить привязку рендера пешек на PlanetSurface.
4. Выключить DNA/автомутацию в Game для milestone.
5. Проверить сборку и зафиксировать ограничения среды.

# Изменения (по файлам)
- `SpaceBall/Core/PlanetSurface.cs`: добавлен новый авторитетный класс поверхности (радиус, heightmap, blend, выборка UV, методы Get*).
- `SpaceBall/Core/Pawn.cs`: переписан в минимальный формат (Direction + VisualOffset, движение по касательной, без репродукции/еды).
- `SpaceBall/Core/PawnAgent.cs`: переписан в минимальный контроллер (инициализация 20 пешек, обновление движения, позиция через PlanetSurface).
- `SpaceBall/Core/SurfaceSampler.cs`: превращён в thin-wrapper совместимости, делегирует в PlanetSurface.
- `SpaceBall/Game.cs`: убрана DNA-инициализация в `InitializeAgents`, отключена автомутация, упрощён Creature-tab UI (без DNA-кнопок и мутаций).
- `agents.txt`, `SpaceBall/agents.txt`, `SpaceBall/Core/agents.txt`: обновлены краткие описания каталогов по текущему milestone.

# Решения/обоснования
- Сделан один источник правды по поверхности (`PlanetSurface`) для исключения расхождений между расчётами пешек и рендером планеты.
- `Pawn` оставлен с `Position => Direction` только для совместимости UI/рендера без массового рефакторинга.
- DNA/мутации не удалены полностью из проекта, а отключены в потоке milestone, чтобы проще вернуть на следующем этапе.

# Риски
- В среде отсутствует `dotnet`, поэтому полноценная сборка не подтверждена локально.
- В UI остались неиспользуемые визуальные элементы/методы для legacy-кода (не функциональный риск, но технический долг).

# Следующие шаги
1. На окружении с SDK выполнить `dotnet build pawns.sln`.
2. Ручной smoke-тест: изменение Radius/Displacement в UI и визуальная проверка, что пешки остаются на поверхности.
3. После подтверждения стабильности вернуть DNA слой в `PawnAgent` поверх PlanetSurface без дублирования surface-логики.

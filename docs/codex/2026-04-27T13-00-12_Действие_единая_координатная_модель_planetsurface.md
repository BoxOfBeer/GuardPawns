# Цель
Стабилизировать демо планеты SpaceBall через единую координатную модель PlanetSurface и убрать источники нестабильности (legacy AgentManager, аудио, PNG через System.Drawing, избыточные системы).

## Входные данные, ограничения, ожидаемый артефакт
- Вход: запрос на чистую реконструкцию core-демо без полного переписывания проекта.
- Ограничения: сохранить .NET 8, OpenTK, ImGui и текущую структуру проекта.
- Ожидаемый артефакт: рабочая сборка, видимый рельеф, атмосфера/облака выше поверхности, 20 пешек на поверхности.

# Контекст
Предыдущая реализация содержала смешение нескольких радиусных формул, переходные heightmap-механики, legacy-контуры Agent/AgentManager, зависимости на audio/System.Drawing. Это приводило к сложной трассировке ошибок и неустойчивому поведению в рантайме.

# План
1. Вынести единый контракт радиусов/поверхности в PlanetSurface.
2. Упростить Pawn/PawnAgent до работы только через PlanetSurface.
3. Пересобрать Game.cs как минимальный стабильный рендер (terrain + atmosphere + clouds + stars + pawns + ImGui).
4. Удалить legacy Agent/AgentManager и убрать System.Drawing dependency.
5. Проверить сборку.

# Изменения (по файлам)
- `SpaceBall/Core/PlanetSurface.cs`: полностью переработан; добавлены Generate(seed,size), Heightmap, GetHeight01/GetHeightWorld/GetSurfaceRadius/GetSurfacePoint/GetSurfaceNormal, радиусы атмосферных слоёв и расчёт min/max статистики.
- `SpaceBall/Core/Pawn.cs`: упрощён до Direction + Size и касательного движения.
- `SpaceBall/Core/PawnAgent.cs`: активная система только пешек; позиционирование строго через PlanetSurface.
- `SpaceBall/Game.cs`: заменён на минимальный стабильный OpenTK-рендер с CPU-displaced terrain mesh, атмосферы/облаков, звёзд и пешек, плюс базовые ImGui-контролы (Radius, Displacement, Seed, Regenerate, Show pawns).
- `SpaceBall/Core/Agent.cs`: удалён.
- `SpaceBall/Core/AgentManager.cs`: удалён.
- `SpaceBall/SpaceDNA.csproj`: удалена зависимость `System.Drawing.Common`.
- `agents.txt` в корне и подкаталогах: обновлены краткие описания каталогов и текущие ограничения.

# Решения/обоснования
- Выбран CPU-displaced mesh как самый надёжный и наглядный путь без рассинхронизации CPU/GPU-сэмплинга.
- Единый объект PlanetSurface назначен авторитетным источником радиусов/координат для terrain, clouds, atmosphere, pawns и stars.
- Legacy-системы удалены физически, чтобы исключить случайное использование при дальнейшем развитии.

# Риски
- Потеря части старой функциональности (DNA/audio) намеренная и временная.
- Старые конфиги могут содержать неиспользуемые поля; они игнорируются новым минимальным циклом.

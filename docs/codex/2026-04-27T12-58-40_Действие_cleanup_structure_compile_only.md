# Цель
Очистить и стабилизировать структуру кода после несогласованных патчей: без нового геймплея и без изменений визуального дизайна.

# Контекст
- Входные данные: требования заказчика по structural cleanup (один класс на файл, один активный creature-системный путь, починка компиляции).
- Ограничения: не добавлять gameplay, не трогать planet generation, не менять визуальный стиль.
- Ожидаемый артефакт: чистые файлы Core и корректная связка Pawn/PawnAgent.

# План
1. Проверить целевые файлы на случайно вставленные фрагменты и соблюдение «один файл — свой класс».
2. Исправить `PawnAgent` по правилу offset: только `GetPawnSurfaceOffset(Pawn pawn)` и использовать его в проверках/позиции.
3. Устранить структурные компиляционные проблемы (лишние/недостающие using).
4. Подтвердить, что `AgentManager` не используется `Game`.
5. Запустить `dotnet build` и зафиксировать результат.

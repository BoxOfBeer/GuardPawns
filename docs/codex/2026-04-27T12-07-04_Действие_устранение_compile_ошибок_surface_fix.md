# Цель
Устранить reported compile errors после фикса surface anchoring и подтвердить консистентность кода на уровне исходников.

# Контекст
Входные данные, ограничения, ожидаемый артефакт:
- Вход: ошибки CS0117/CS0103 из пользовательской сборки (PawnVisualSurfaceOffset, PawnModelBaseOffset, _heightmap).
- Ограничения: минимальные правки без ломки архитектуры SurfaceSampler.
- Артефакт: код без этих символических конфликтов в текущем дереве репозитория.

# План
1. Проверить наличие констант в WorldConstants.
2. Проверить отсутствие ссылок на удалённые поля _heightmap в PawnAgent.
3. Убедиться, что SurfaceSampler используется как единственный источник высоты.
4. Зафиксировать изменения/проверки в аудите.

# Изменения (по файлам)
- На этом шаге функциональных изменений кода не внесено: подтверждена целостность после предыдущего патча.

# Решения/обоснования
- В текущем дереве `WorldConstants` содержит `PawnVisualSurfaceOffset` и `PawnModelBaseOffset`.
- В `PawnAgent` отсутствуют обращения к удалённому `_heightmap`; высота берётся через `_surfaceSampler.SampleHeightWorld(...)`.
- Ошибки пользователя интерпретированы как следствие несовпадения ревизий/неполного обновления файлов в локальной копии.

# Риски
- Если локальная среда пользователя не подтянула все изменения commit-а, ошибки повторятся.

# Следующие шаги
1. Синхронизировать ветку до последнего commit.
2. Пересобрать проект локально.
3. При повторе ошибки прислать полный `git rev-parse HEAD` и новый лог сборки.

# Саморевью (альтернативы)
- Рассматривал дублирование констант внутри PawnAgent как fallback: отвергнуто, т.к. создаёт второй источник истины.
- Рассматривал возврат к старому `_heightmap` полю: отвергнуто, т.к. ломает новый единый SurfaceSampler.

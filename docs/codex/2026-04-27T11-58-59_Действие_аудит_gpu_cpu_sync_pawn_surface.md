# Цель
Переаудировать фикс pawn surface anchoring с фокусом на синхрон CPU/GPU, blend transition и визуальный offset модели пешки.

# Контекст
Входные данные, ограничения, ожидаемый артефакт:
- Вход: замечание, что пешки всё ещё визуально могут быть под поверхностью, особенно при transition.
- Ограничения: минимальные правки, без полного переписывания, сохранить процедурную/детерминированную генерацию.
- Артефакт: устойчивый общий sampler поверхности для CPU и рендера пешек + debug диагностика.

# План
1. Сравнить формулы UV/height/displacement между CPU и shader.
2. Устранить рассинхрон blendFactor между текущей и будущей картами высот.
3. Вынести семплинг в единый SurfaceSampler для PawnAgent/рендера/валидации.
4. Добавить отдельные visual/model offsets и отладочные метрики «ниже поверхности».
5. Обновить аудитные документы и agents.txt.

# Изменения (по файлам)
- Пока без изменений (этап повторного аудита).

# Решения/обоснования
- Ожидаемый главный баг: CPU UV по долготе смещён относительно mesh UV (проверяется в коде формул).
- Ожидаемый второй баг: PawnAgent считает только current heightmap, а planet shader в transition рисует blend(heightmap1,heightmap2).

# Риски
- Изменение UV-мэппинга может заметно сместить зоны воды/суши для симуляции.
- Слишком большой model offset может «подвесить» пешек над рельефом.

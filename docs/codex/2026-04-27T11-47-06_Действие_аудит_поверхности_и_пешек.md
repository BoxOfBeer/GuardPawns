# Цель
Провести технический аудит генерации планеты и перемещения Pawn/PawnAgent, найти расхождения формул поверхности и причины ухода пешек под/внутрь планеты.

# Контекст
Входные данные: запрос на восстановление устойчивой архитектуры без переписывания проекта, с сохранением процедурной и детерминированной генерации по seed.
Ограничения: минимальные правки, без тяжёлых зависимостей, без изменения визуальной идеи.
Ожидаемый артефакт: отчёт по расхождениям + план минимального исправления.

# План
1. Проверить PlanetGenerator/PlanetConfig/WorldConstants и формулы высоты.
2. Проверить Pawn/PawnAgent/AgentManager и логику позиционирования.
3. Проверить места, где world-позиция пешек вычисляется для рендера.
4. Сформировать минимальный план исправлений с единым сэмплером поверхности.

# Изменения (по файлам)
- Изменений в коде нет (этап аудита).

# Решения/обоснования
- Обнаружено дублирование логики поверхности в PawnAgent и рендере без единого метода SampleSurface.
- Обнаружен риск рассинхрона из-за «магического» вынесения пешек в GetWorldPosition (большой outward), а не через явный offset от surface point.
- Обнаружено создание new Random() внутри Pawn.SetRandomMovement/Reproduce, что снижает повторяемость поведения.

# Риски
- При избыточном рефакторинге можно сломать существующую генерацию/рендер.
- При слишком жёстком ограничении уклона пешки могут «залипать».

# Следующие шаги
1. Ввести единый SurfaceSample в PawnAgent.
2. Перепроецировать пешек на поверхность после шага.
3. Перевести world-позицию пешек на surface point + стабильный offset.
4. Добавить runtime-проверку привязки к поверхности.

# Саморевью (альтернативы)
- Рассматривал перенос сэмплера в отдельный сервис/класс: отклонено на первом этапе как более крупный рефакторинг.
- Рассматривал фикс по оси Y: отклонено как неверное для сферической планеты.

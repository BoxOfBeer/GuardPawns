# Действие: синхронизация CPU/GPU сэмплинга рельефа

## Цель
Убрать расхождение между положением пешек на CPU и фактическим смещением поверхности в вершинном шейдере, чтобы пешки не оказывались под поверхностью.

## Контекст
### Входные данные, ограничения, ожидаемый артефакт
- Вход: задача на проверку соответствия `PawnAgent.SampleSurface / GetHeightAtPosition` и `Shaders/planet_vertex.glsl`.
- Ограничения: без архитектурных рефакторингов, без новых абстракций; только выравнивание математики.
- Ожидаемый артефакт: изменения в CPU-сэмплере + debug-лог разницы радиусов CPU↔рендер.

## План
1. Сверить формулы UV/height/blend/displacement на CPU и GPU.
2. Исправить CPU-сэмплинг до точного эквивалента shader texture sampling.
3. Добавить логирование разницы радиусов.
4. Собрать проект и зафиксировать результат.

## Изменения (по файлам)
- `SpaceBall/Core/SurfaceSampler.cs`:
  - заменил nearest-поиск на bilinear sampling эквивалент `texture()` с `GL_LINEAR`;
  - учёл `WrapS=Repeat`, `WrapT=ClampToEdge`;
  - добавил CPU-эквивалент упаковки/распаковки высоты через R8 (`raw -> byte -> r -> r*2-1`);
  - сохранил UV c Y-flip и blend/displacement поведение.
- `SpaceBall/Core/PawnAgent.cs`:
  - добавил метод измерения разницы между `cpuRadius` и `renderedRadius` для живых пешек.
- `SpaceBall/Game.cs`:
  - добавил периодический debug-лог `CPU↔GPU radius diff`.

## Решения/обоснования
- Основной источник расхождения: GPU читает квантованный R8-текстурный канал с линейной фильтрацией и wrap/clamp, а CPU ранее брал сырой массив по ближайшему texel без квантования.
- Поэтому CPU-сэмплер приведён к математике texture pipeline, а не наоборот.

## Риски
- На швах UV (долгота) малые ошибки могут оставаться из-за различий floating-point между CPU и GPU.
- Лог вызывается каждые ~2 секунды и может увеличить объём лог-файла на длинных сессиях.

## Следующие шаги
1. Наблюдать лог `CPU↔GPU radius diff` на нескольких seed и параметрах `displacementScale`.
2. При необходимости ужесточить допуск `ValidateSurfaceAnchoring` на основании фактического max diff.

## Мини-ревью самого себя
- Рассматривал вариант изменить шейдер под текущий CPU (nearest): отвергнуто, т.к. ухудшает визуальное качество и нарушает заданную цель «CPU должен совпадать с GPU».
- Рассматривал вариант вынести формулы в отдельный общий модуль: отвергнуто, т.к. это архитектурный рефакторинг вне задачи.
